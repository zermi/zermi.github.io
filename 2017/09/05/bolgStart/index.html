前言

本人很幸运的赶上了公司的转型，从传统的jq转向模块化开发。
由于公司研发力量有限（全公司人数700+），老项目众多，许多老人无法脱身，所以本人有幸一直参与UI库的各项工作。研发过程中，发现之中也发现了很多问题，打算一一总结下来，以备日后逐一优化，不断学习，如果能为他人提供帮助自然更好。由于革命尚未完成，会持续更新。

一、样式问题

1.1 避免使用内联样式

目前基本组件的编写已经基本结束，只剩下一些如富文本编辑器，文章等稍复杂一些的组件交给老人去编写。本人被分配到编写学习演示网页的工作，在编写演示组件模块时，发现了这个问题。
为了让使用者清晰明了的了解组件的属性接口，我采用演示效果配合代码的演示方式。
在演示过程中发现，原组件的默认样式对使用造成很大影响，当然我们会提供类名接口和style接口以便于大家修改样式，不过这显然造成了一些重复工作。但这不是我们讨论的重点，重点在于我们的组件中少部分采用了内联样式，包括利用js判断之后添加的样式，比如说菜单箭头位置利用绝对定位，这样就局限了使用中样式必须是绝对定位才能奏效。同时内联样式优先级极高，外部无法覆盖，会对使用造成很大的困难。

1.2 外层避免采用固定值

这里的固定值并不是指px为单位的值，包括”10rem 20rem”等，可能在某些组件内部布局时出于开发时效果明显或者某些布局比例等因素采用了这种布局，但是这样的设置在外部使用时可能会造成问题。
这里我比较推荐外层采用”width: 100%;height: 100%”,这样把它的宽高交给使用它的容器，既比较容易控制，同时不宜出现冲突。不过要注意，这样调用组件时它可能由于外容器没有高度而是隐藏的，不要因此而忘记了它。
